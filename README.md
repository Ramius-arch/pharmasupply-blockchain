### 4. Running the Full Stack

There are two ways to run the application:

#### Option 1: Using the Application Manager (Recommended)
A CLI-based GUI is provided to easily manage all the services.

To start the manager, run the following command from the root of the project:
```bash
npm start --workspace=manager
```
This will open an interactive menu that allows you to:
- Start and stop all services at once.
- Start and stop individual services (Hardhat, Backend, Frontend).
- Deploy the smart contract.
- Seed the database.
- View the status of all services.

#### Option 2: Running Services Manually

##### Step 1: Start the Local Blockchain
In a new terminal, navigate to the `web3` directory and start the Hardhat local network:
```bash
cd web3
npx hardhat node
```
Keep this terminal open. All blockchain transactions will be logged in this terminal.

##### Step 2: Deploy the Smart Contract
In another new terminal, navigate to the `web3` directory and deploy the smart contract:
```bash
cd web3
npx hardhat run scripts/deploy.js --network localhost
```
This will create a `contract-address.json` file in the `web3` directory. The backend will automatically use the address and ABI from this file.

##### Step 3: Run the Demo Transaction Generation Script (after contract deployment)
In another new terminal, navigate to the `web3` directory and run the script to generate approximately 300 demo transactions:
```bash
cd web3
npx hardhat run scripts/generate-demo-transactions.js --network localhost
```
This script will create 150 new products and update the status of 150 of these products on the blockchain. It now correctly parses events for Ethers.js v6. You should see logs of transaction hashes and item IDs/statuses being created/updated in the Hardhat node terminal.

##### Step 4: Seed the Database
In another new terminal, navigate to the `backend` directory and run the seeding script:
```bash
cd backend
npm run seed
```
**Note on Product Images**: The seed script currently populates products with placeholder images from `picsum.photos`. To fully realize the "pharmaceutical ad with Kodak film feel" theme, you will need to replace these placeholder URLs in `backend/seeders/seed.js` with URLs of your chosen themed images. Remember to re-run `npm run seed` and restart your backend after making changes to `seed.js`.

##### Step 5: Start the Backend and Frontend
In another new terminal, from the root of the project, run:
```bash
npm run dev
```
The backend will run on `http://localhost:3000` and the frontend on `http://localhost:5173`.

### Viewing Blockchain Transactions

You can view blockchain transactions in a couple of ways:

1.  **Hardhat Node Terminal**: The terminal where you ran `npx hardhat node` (Step 1 of Option 2) will continuously log all transactions that occur on your local blockchain network. This includes the transactions generated by `generate-demo-transactions.js` and any transactions initiated by the backend (e.g., when creating new products via the Admin Dashboard). You will see transaction hashes, gas usage, and event emissions.

2.  **Local Blockchain Explorer (Recommended for detailed view)**: For a more user-friendly and detailed view of transactions, blocks, and smart contract interactions, you can set up a local blockchain explorer like Blockscout.
    *   You would typically run Blockscout (or an Etherscan-like explorer) in a Docker container and configure it to connect to your Hardhat node's RPC URL (`http://localhost:8545`).
    *   This provides a web interface where you can search by transaction hash, block number, address, and view event logs. Instructions for setting up Blockscout with a local Hardhat network are typically found in Blockscout's official documentation.

### How to Use the Application

1.  **Create an Account:**
    -   Navigate to `http://localhost:5173/register` and create a new account.
2.  **Log In:**
    -   Go to `http://localhost:5173/login` and log in with your newly created account.
3.  **Make a Purchase (Mock Payment)**:
    -   Add products to your cart.
    -   Proceed to `http://localhost:5173/checkout`.
    -   Fill in shipping details (these are currently not validated).
    -   Click "Review Order & Pay" to simulate a successful payment.
    -   Upon simulated successful payment, the order will be processed, and you'll be redirected to a success page.
4.  **Generate a Blockchain Wallet:**
    -   Navigate to `http://localhost:5173/generate-wallet`.
    -   Click the "Generate Wallet" button to create a new Ethereum address and private key.
    -   **IMPORTANT**: Securely store your generated private key offline. Do not share it, as loss of a private key means loss of funds.
5.  **Create a Product:**
    -   Go to the Admin Dashboard (`http://localhost:5173/admin`).
    -   In the "Product Management" section, fill out the form to create a new product.
    -   When you create a product, a corresponding item will be created on the blockchain.
6.  **View Product and Blockchain History:**
    -   Go to the homepage or the products page and click on a product to view its details.
    -   On the product details page, you will see the blockchain information, including the item ID, status, and history.
7.  **Submit a Product Review**:
    - Navigate to a product's detail page.
    - You can now input a rating and a comment in the "Customer Reviews" section and submit your review.

    ```bash
    cd e2e-tests
    ```
3.  Run the script to create 13 mock products:
    ```bash
    npm run create-products
    ```
    This will create 13 products and save their data to a `products.json` file.
4.  Run the script to verify the created products:
    ```bash
    npm run verify-products
    ```
    This will read the `products.json` file, fetch the data for each product from the backend API, and verify that it matches.
